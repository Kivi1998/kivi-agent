## LLM Provider 当前方案（核心结论）

**结论：daemon 不需要改任何代码即可切换 Anthropic 兼容端点。**

| 场景 | 配置 | 行为 |
|---|---|---|
| 真实 Anthropic（默认） | `ANTHROPIC_API_KEY=sk-...`（不设 `ANTHROPIC_BASE_URL`） | 走 `https://api.anthropic.com/v1/messages` |
| Anthropic 兼容的第三方（DeepSeek / Moonshot 等） | `ANTHROPIC_API_KEY=sk-...` + `ANTHROPIC_BASE_URL=https://<host>/anthropic` | `anthropic` Python SDK 0.105.2 原生读 `ANTHROPIC_BASE_URL`，base url 透明替换 |
| 模型名 | `KAMA_LLM_DEFAULT_MODEL=deepseek-v4-pro`（TOML 或环境变量） | 写入 `KamaConfig.llm.default_model` → 写入 HTTP body `model` 字段 |

**注意**：`core/llm/factory.py` 同时支持 `provider=openai_compat`（OpenAI 协议路径），用于走 `/v1/chat/completions` 的端点（如 Moonshot 原生 OpenAI）。两条路径并存，按 `llm.provider` 字段路由。

### TUI 状态栏可见性

`kama-tui` 顶部状态栏实时显示 `provider/model`（commit `38f6b8f`），用户在终端可一眼确认当前 LLM 配置：
- `anthropic/claude-sonnet-4-6`（默认）
- `openai_compat/kimi-k3`（Moonshot）
- `anthropic/deepseek-v4-pro`（DeepSeek /anthropic 兼容端点）
