**唯一权威**：C 报告锁定的 6 个顶层 Tool 名。所有 Skill `allowed_tools` / Profile `allowed_tools` / 前端 Tool 名称 / 协议命令参数都按此对齐。

| `rag_query` | RAG 检索（含**内部** rewrite + retrieval） | `rag_query_rewrite`（B 误为独立 Tool；C 已合入内部） | C §业务 Tool 迁移矩阵 3.3 + 3.4 |

### 规则

1. **不可再加 `*_v2` 之类的副名**。若需新 Tool，名字加进 v2 契约（走 ADR）。

3. **冻结的 6 个名字与 B 报告中的占位**（`search_knowledge_base` / `rag_query_rewrite` / `note_save` / `recall_memory` / `db_query` / `chart_render`）全部弃用。B 报告里 6 处提及旧名的地方在 Wave 1 开工前需批量替换。

## 2. RunContext v1 字段表（冻结）

**唯一权威字段集**。A / B / C / D / E 各自列出的 `datasource_id` vs `db_id`、`original_query` vs `user_query`、`runtime_metadata` vs `runtime_values`、`frontend_connection_id` vs `extra_tools` 等差异全部合并到下列统一字段。

```python
@dataclass
class RunContext:
    """运行参数上下文。Wave 1 起按本字段表冻结。

    不在本表中的字段视为非法。A 报告提议的 schema_version 字段作为
    dataclass 字段保留，便于未来 v2 升级时做兼容性检查。
    """
    schema_version: int = 1              # 必填，默认 1
    run_id: str                          # 必填，每次 run 唯一
    trace_id: str                        # 必填，关联 Trace
    user_id: str                         # 必填，租户隔离
    session_id: str                      # 必填，会话隔离
    datasource_id: str | None = None     # 可选，问数 Tool 由 C 注入
    knowledge_base_id: str | None = None # 可选，RAG Tool 由 C 注入
    frontend_connection_id: str | None = None  # 可选，Web 前端连接 ID
    runtime_values: dict[str, Any] = field(default_factory=dict)  # 灵活扩展
```

### 字段合并映射

| 旧名（被弃） | 新名（冻结） | 出处 |
|---|---|---|
| `db_id` | `datasource_id` | C §3.8 |
| `user_query` / `original_query` | **不在 RunContext** | 属于消息上下文（`ExecutionContext.messages`），不在运行参数里重复 |
| `conversation_history` | **不在 RunContext** | 同上 |
| `runtime_metadata` / `extra_tools` | `runtime_values` | A §1, D §1 |
| `frontend_connection_id` | `frontend_connection_id` | 名字统一，无变化 |

### 不在 RunContext 的字段

以下字段是**消息上下文**或**执行上下文**的一部分，不在 RunContext：

- 用户原始问题 / 会话历史 → `ExecutionContext.messages`（已有）
- agent_name / parent_run_id / team_id / route_history → `ExecutionContext` 扩展（见 §3.3 A 报告）

### 升级到 v2 的流程

任何 Agent 需要在 RunContext 加字段：

1. 写 ADR：`docs/adr/0001-runcontext-v2.md`，说明新字段、迁移路径、向后兼容策略
2. 主控 Agent 评审 + 合并
3. `schema_version` 字段从 1 → 2
4. Wave 1 旧代码按 ADR 兼容策略处理

### 6.3 E 报告 Judge 成本修正

E 报告原文（line 704-705）：

> "单 case 1k 输入 + 0.5k 输出 ≈ $0.0000015。1000 cases ≈ $0.0015，可忽略。"

**算错了 1000 倍**。正确计算：

```
单价 = $0.001 / 1k tokens
单 case tokens = 1.5k（1k 输入 + 0.5k 输出）
单 case 成本 = $0.001 × 1.5 = $0.0015
1000 cases 总成本 = $0.0015 × 1000 = $1.50
```

**正确结论**：1000 cases 约 $1.50，不是 $0.0015。绝对值仍然很低（演示版可忽略），但 E 报告的数字**少算了 1000 倍**，Wave 1 启动前必须更正。

E 报告提交时未自检。Wave 1 启动时主控 Agent 会向 E 反馈此错误，让 E 在该报告正式合并前修订。
